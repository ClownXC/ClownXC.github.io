phoenix&hbase集成
发表于2019-10-18|更新于2021-05-25|滴滴智慧出行
|字数总计:2.7k|阅读时长:14分钟|阅读量:

Apache Phoenix 是构建在 HBase 之上的关系型数据库层，作为内嵌的客户端 JDBC 驱动用以对 HBase 中的数据进行低延迟访问。Apache Phoenix 将用户编写的 sql 查询编译为一系列的 scan 操作，最终产生通用的 JDBC 结果集返回给客户端。数据表的元数据存储在 HBase 的表中被会标记版本号，所查询的时候会自动选择正确的schema。直接使用 HBase 的API，结合协处理器(coprocessor)和自定义的过滤器，小范围的查询在毫秒级响应，千万数据的话响应速度为秒级。

1. Phoenix 简介

Phoenix 是 HBase 的开源 SQL 中间层，它允许你使用标准 JDBC 的方式来操作 HBase 上的数据。在 Phoenix 之前，如果你要访问 HBase，只能调用它的 Java API，但相比于使用一行 SQL 就能实现数据查询，HBase 的 API 还是过于复杂。Phoenix 的理念是 we put sql SQL back in NOSQL，即可以使用标准的 SQL 就能完成对 HBase 上数据的操作。同时这意味着你可以通过集成 Spring Data JPA 或 Mybatis 等常用的持久层框架来操作 HBase.
Phoenix 查询引擎会将 SQL 查询转换为一个或多个 HBase Scan，通过并行执行来生成标准的 JDBC 结果集。它通过直接使用 HBase API 以及协处理器和自定义过滤器，可以为小型数据查询提供毫秒级的性能，为千万行数据的查询提供秒级的性能。同时 Phoenix 还拥有二级索引等 HBase 不具备的特性。

5.拷贝核心jar包

通过上面的简单介绍，我们知道了phoenix是作为hbase的一个jdbc驱动对外提供jdbc的支持.这样一来我们就需要讲phoenix核心的jar包拷贝到hbase所有的节点(regionserver)的lib目录下.

这里还有个坑需要大家注意,看下面官网的解释

官网解释

Installation

To install a pre-built phoenix, use these directions:

  • Download and expand the latest phoenix-[version]-bin.tar.

    看下面这句话，如果是phoenix 3.x版本的安装除了phoenix-[version]-server.jar需要复制外，同时phoenix-core-[version].jar 也需要复制，但是我们安装的是phoenix 4.x所以phoenix-core-[version].jar 的jar包就不需要复制了

  • Add the phoenix-[version]-server.jar to the classpath of all HBase region server and master and remove any previous version. An easy way to do this is to copy it into the HBase lib directory (use phoenix-core-[version].jar for Phoenix 3.x)

  • Restart HBase.

  • Add the phoenix-[version]-client.jar to the classpath of any Phoenix client.

将【PHOENIX_HOME】目录下的phoenix-[version]-server.jar添加到HBase集群的所有regionserver节点的lib目录下，然后重启HBase集群。
